## Some Important Java Concepts

### Java Virtual Machine
The Java Virtual Machine (JVM) is an abstract machine that executes Java bytecode and provides a platform-independent environment for running Java programs.
- JVM is the engine that runs the Java bytecode.
- When you compile Java code, it becomes bytecode (.class files).
- JVM reads the bytecode and executes it on your operating system.<br>

#### Main responsibilities
- Executes Java programs
- Converts bytecode into machine code
- Manages memory (Garbage Collection)
- Provides platform independence ("Write Once, Run Anywhere")
- Handles security and thread management

#### Simple flow

Java Code (.java) --> Compiled by javac --> Bytecode (.class) --> Runs on JVM

## JRE - Java Runtime Environment
The Java Runtime Environment (JRE) is a software package that provides the libraries, Java Virtual Machine (JVM), and other components required to run Java applications.

## JDK (Java Development Kit)
The JDK is a software development package that provides tools required to develop, compile, debug and run Java applications. It includes the JRE and development tools such as the Java compiler (javac)

### Java Bytecode
Java bytecode is the intermediate, platform-independent code generated by the Java compiler from source code, which is executed by JVM.

### Java Compiler (javac)
The Java compiler (javac) is a tool provided in the JDK that converts Java Source code (.java files) into bytecode (.class files).
